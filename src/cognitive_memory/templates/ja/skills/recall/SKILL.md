# SKILL: /recall — 作業記録横断検索

**目的**: 全記憶ソースを横断検索し、過去の文脈を復元する

`/search` が cogmem のセマンティック検索のみなのに対し、`/recall` は 3 つのソースを検索する。
プロジェクトのパス、設定、URL など構造化情報は auto-memory に、実装の詳細はログの非インデックス部分に記録されていることが多い。cogmem のセマンティック検索だけでは見つからない情報をカバーするのがこのスキルの価値。

---

## 実行手順

### Step 1: キーワード抽出

- `/recall [キーワード]` の場合: そのまま使用
- 自動トリガーの場合: 会話から「何がわからないのか」を特定し、3〜5 キーワードを抽出

### Step 2: auto-memory を最初に検索する（高速・高精度）

auto-memory はプロジェクト設定・パス・URL・手順など構造化情報の宝庫。まずここを調べる。

```
Grep tool:
  pattern: [キーワード1|キーワード2|キーワード3]
  path: ~/.claude/projects/<auto-detected-project>/memory/
  output_mode: content
  context: 3
```

MEMORY.md（インデックス）も検索対象に含める。ヒットしたファイルは Read して全文を確認する。

**早期リターン判定**: auto-memory で質問に対する明確な回答（URL、パス、設定値、手順）が見つかった場合、Step 3〜4 をスキップして直接回答してよい。これによりトークン消費と応答時間を大幅に節約できる。

### Step 3: memory/logs/ を直接 grep する

cogmem のセマンティック検索はログエントリ（`###` で始まるブロック）のみインデックスしている。ログの「## 実装内容」「## 引き継ぎ」などのセクションはインデックス外のため、grep で直接検索する必要がある。

```
Grep tool:
  pattern: [キーワード1|キーワード2|キーワード3]
  path: memory/logs/
  output_mode: content
  context: 5
```

ヒットがあれば、該当ファイルの関連セクションを Read して詳細を取得する。

### Step 4: cogmem セマンティック検索（補完）

Step 2〜3 で十分な情報が得られなかった場合のみ実行する。

```bash
cogmem search "[キーワード]" --json
```

意思決定の経緯や洞察など、キーワードでは引っかかりにくい意味的に関連する情報を探す。

### Step 5: 結果の統合と回答

検索結果をマージし、以下の優先順で提示:

1. **auto-memory のヒット**（構造化情報: パス、URL、設定値など即座に使える情報）
2. **ログ grep のヒット**（実装詳細、引き継ぎ、セッション概要）
3. **cogmem の高スコア結果**（score >= 0.75: 意思決定の経緯、洞察）

出力フォーマット:

```markdown
## /recall 結果: "[キーワード]"

### auto-memory
- **[ファイル名]**: [関連する内容の抜粋]

### ログ
- **[日付]**: [関連する内容の抜粋]

### 回答
[検索結果を統合した回答。「見つからなかった」場合はその旨を伝える]
```

ヒット件数が少ない場合や単純な質問への回答では、テーブルやセクション分けは省略して簡潔に答えてよい。

## 表現ルール

- 覚えている体で自然に回答すること
- 禁止: 「検索する」「確認する」「履歴を調べる」等のシステム的表現
- 必ず検索してから「わからない」と答えること

---

## 自動トリガー条件

以下の状況を検知したら、ユーザーに聞かずにこのスキルを実行する:

- ファイルパスやプロジェクトの場所がわからない
- 「前にやった」「以前の」作業の詳細が必要
- 設定値、URL、トークンの保存場所が不明
- `/search` で cogmem を検索したがヒットしなかった、または不十分だった
- ユーザーが「過去ログ見て」「ログから探して」「ログを確認して」「メモリを見て」「前にやったはず」「以前の調査見て」と言った

## `/search` との使い分け

| | `/search` | `/recall` |
|--|-----------|-----------|
| 検索対象 | cogmem（memory/logs/ エントリのみ） | auto-memory + ログ全文 + cogmem |
| 得意な情報 | セッションの洞察・決定・パターン | プロジェクト設定・パス・URL・実装詳細 |
| 使い分け | 過去の議論や思考を振り返る | 具体的な情報を探す・文脈を復元する |
| 速度 | 速い（cogmem 1回） | 必要に応じて段階的（早期リターンあり） |
