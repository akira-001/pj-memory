# Insights

## INS-001: ダッシュボード UI はデザイン確定後に実装すべき
- **日付**: 2026-04-04
- **ドメイン**: dashboard, UI
- **確信度**: Established（6回の修正で裏付け）
- **内容**: UI コンポーネント（トグルスイッチ、カードレイアウト等）の実装前にデザインを確定しないと、fix コミットが連続する。デザイン先行フローを徹底すると手戻りを削減できる。

## INS-002: Claude Code フックイベントのない機能は最近傍イベントで代替する
- **日付**: 2026-04-11
- **ドメイン**: claude-code, hooks, architecture
- **確信度**: Established
- **内容**: Claude Code には「context compression 直前」のような特定ライフサイクルイベントが存在しないことがある。その場合、「コンテキスト圧縮を引き起こす操作」（= サブエージェント起動 = Task ツール使用）を PreToolUse でフックすることで実質的に同等の効果を得られる。イベントが存在しないと即座に諦めず「何がそのイベントを引き起こすか」を考えること。

## INS-003: モッキング対象は公開 API でなく実際に呼ばれる内部関数を選ぶ
- **日付**: 2026-04-11
- **ドメイン**: testing, mocking
- **確信度**: Established（2件のテスト失敗で裏付け）
- **内容**: `shutil.which` をモックしても、テスト対象が独自の `_find_ollama()` 関数を経由していると効果がない。モック対象は「テスト対象モジュール内でインポートされている名前空間」（`cognitive_memory.dashboard.services.ollama_service._find_ollama`）を指定すること。
